# Agent Module — Worker-Driver Architecture

Hierarchical planning and control system for ARCPro intersection navigation. Replaces the hardcoded `set_turn_bias()` pattern with graph-based route planning, decentralized multi-agent coordination, and topological localization.

## Architecture

Each vehicle is managed by an **AgentNode** containing two internal nodes:

```
AgentNode
├── Worker  (classical planner — WHERE to go)
│   ├── Reads position from PhysX (training) / topological EKF (deployment)
│   ├── Consults intersection graph for available exits
│   ├── Picks turn: LEFT / STRAIGHT / RIGHT
│   ├── Broadcasts intent for multi-agent coordination
│   └── Returns (turn_token, go_signal) -> injected into obs vec[0:2]
│
└── Main    (learned controller — HOW to get there)
    ├── Injects Worker output into observation before policy sees it
    ├── Policy forward pass: CNN+LSTM+PPO conditioned on turn_token
    └── Safety gate: if go_signal == 0, override throttle=0 brake=1
```

The **WorkerScheduler** runs locally on each robot (decentralized). It listens for other robots' broadcast intents over ROS2 and returns go/wait based on conflict detection and RVO time-gap analysis.

### Nested Action Loop

```
OUTER (Worker decides direction):
    Worker queries graph -> turn_token {-1, 0, 1}
    turn_token injected into vec[0] -> conditions kinematic anchors

    INNER (Main decides go/brake):
        Scheduler checks conflicts -> go_signal {0.0, 1.0}
        go_signal == 0 -> BRAKE (safety override)
        go_signal == 1 -> policy controls throttle/brake (learned)
```

### Intersection Phases

**Approaching** (Worker: CRUISING) — EKF tracks arc-length `s` along current edge. Worker holds last turn_token. Policy drives using caeera + stable token. Trigger fires when `s >= edge_length - threshold`.

**At intersection** (Worker: DECIDING -> COMMITTED) — Worker picks a turn from available exits, broadcasts intent, checks for conflicts. Turn token locks and stays stable through the entire maneuver. `vec[0] = turn_token`, `vec[1] = go_signal`.

**Through intersection** (Worker: COMMITTED -> CRUISING) — Policy executes the turn with kinematic anchors curved in the chosen direction. EKF transitions to the outgoing edge: `s` resets to 0, covariance collapses, all accumulated drift is killed. Cooldown prevents re-triggering.

## Topology vs Geometry

**Topology** is human-authored and stable — it defines which roads exist and which turns connect them. Lives in `config/intersection_topology.json`. No positions, no lengths, no coordinates. Only changes when roads are added or removed.

**Geometry** is auto-discovered and ephemeral — intersection positions, edge lengths, exact headings. Generated by `GeometryCalibrator` from PhysX ground truth during training, saved to `config/geometry_cache.json`. When the USD map changes, delete the cache and re-run.

## Telemetry Vector Protocol

The 12-float observation vector `vec` is updated in two slots:

| Index | Name | Old | New |
|---|---|---|---|
| 0 | `turn_token` | `turn_bias` (manual float) | `{-1, 0, 1}` from Worker |
| 1 | `go_signal` | `reserved` (always 0.0) | `{0.0, 1.0}` from Scheduler |
| 2–11 | — | unchanged | unchanged |

```python
from agent import (
    AgentEnvWrapper, AgentConfig, WorkerConfig,
    IntersectionGraph, GeometryCalibrator, CalibrationConfig,
)

def _init():
    env = IsaacDirectEnv(config=config)

    graph = IntersectionGraph.from_json("config/intersection_topology.json")

    calibrator = GeometryCalibrator(graph, CalibrationConfig())
    if not calibrator.try_load_cache():
        pass  # Will calibrate incrementally during training

    agent_config = AgentConfig(
        agent_id="agent_0",
        worker=WorkerConfig(mode="curriculum"),
    )
    env = AgentEnvWrapper(env, graph=graph, agent_config=agent_config)

    if use_waypoint_wrapper and HAS_WAYPOINT_WRAPPER:
        env = WaypointTrackingWrapper(env, env_id=0)
    env = Monitor(env)
    return env
```

## When the Map Changes

1. Modify the USD scene
2. Delete `config/geometry_cache.json`
3. Run training — geometry auto-calibrates from PhysX
4. Cache regenerates automatically

No JSON editing. No manual measurement.

## Multi-Agent (future, decentralized)

Each robot runs its own AgentNode locally on its Intel NUC. Coordination happens through lightweight ROS2 topic broadcasts:

```python
# In simulation (shared process, one scheduler):
scheduler = WorkerScheduler()
env_0 = AgentEnvWrapper(base_env_0, graph, AgentConfig("car_0"), scheduler)
env_1 = AgentEnvWrapper(base_env_1, graph, AgentConfig("car_1"), scheduler)

# On real robots (separate NUCs, ROS2 broadcast):
# Each NUC runs its own Agent + local conflict checker
# Intent messages published/subscribed over /agent_intents topic
```
